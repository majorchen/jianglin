# 降临 · 手机端 HD-2D 升级方案

> 2026-06-10 制定。目标：从"手机可看的原型"升级为"第一眼想截图发群里的 HD-2D 手机端活世界"。
> 风格基准：八方旅人（Octopath Traveler）——低分辨率像素 sprite × 3D 深度 × 电影级后处理。

---

## 一、诊断：手机端为什么"稀烂"（实测 2026-06-10，390pt 视口）

| # | 问题 | 根因 | 严重度 |
|---|------|------|--------|
| 1 | 所有 UI 文字读不清 | 设计分辨率 720×1280 + `canvas_items/expand` 缩放，iPhone 390pt 宽时全 UI 缩到 **0.54 倍**，13-14px 字号实际只有 ~7pt | 🔴 根因级 |
| 2 | 首屏被文字条吃掉 | 顶部同时叠 4 层横条（标题/资源/全局台词/简报），事件再弹一层，世界只剩 60% 屏幕 | 🔴 |
| 3 | 按钮难点中 | 底部按钮有效高度 ~23pt（< 44pt 最低标准）；点击判定 80px 半径，无 UI 事件拦截，误触率高 | 🔴 |
| 4 | 节奏失控 | `time_speed=0.18` → 一天 133 秒快进循环，台词/事件刷屏，与"日更世界"定位冲突 | 🟡 产品级 |
| 5 | 没有 HD-2D 质感 | `no_depth_test` + billboard → 无遮挡关系；无影子；无景深/光晕；地面单张 1024px 纹理平铺 + 两条 BoxMesh 路 | 🟡 |
| 6 | 资产拼贴感 | 尺寸/像素密度完全不统一：b0=287×583、b1=326×336、角色帧≈313px，调色板各自为政，靠代码 scale 硬凑 | 🟡 |

结论：架构方向是对的（Godot 3D + Sprite3D billboard 正是 HD-2D 的正确底座），问题集中在 **分辨率策略、信息架构、资产规范** 三件事，全部可修。

## 二、目标画面定义

竖屏一屏 = 一张可以直接截图发社群的"电影感小镇"：

- 像素角色走在有真实遮挡的 3D 小镇里，脚下有影子
- 屏幕上下 25% tilt-shift 渐进虚化（八方旅人标志性镜头）
- 暖色平行光 + 长影子 + 夜晚建筑窗户亮灯，昼夜随北京时间流转
- 首屏只有：世界 + 一条状态条 + 一个底部 dock，其余全部收进 bottom sheet

## 三、资产规范与管线（P1）

### 3.1 规格标准（先定标准，再生成）

**像素密度锚：1 世界单位 = 48px**（全项目唯一标准，杜绝 scale 硬凑）

| 资产 | 规格 | 说明 |
|------|------|------|
| 角色 | 每帧 48×72，4×4 sheet = 192×288 | 行序：idle / 下 / 侧 / 上 |
| 小型建筑（厨房/掩体） | 144×144~192（3 格底座） | 同一相机角度（俯视 ~30°） |
| 中型建筑（温室/瞭望台） | 192×192~240（4 格底座） | |
| 高建筑（广播塔） | 144×288 | |
| 地形 tile | 48×48 单 tile，16-24 个变体 | 土路/碎石/草丛/裂缝/边缘过渡 |
| 调色板 | 全项目统一 48 色 | 废土暖棕主调 + 锈红点缀，PIL quantize 统一 |

### 3.2 生成管线（写成 `scripts/asset_pipeline.py`，固定 4 步）

1. **生成**：GPT Image 2（codex_gen_cli.py），prompt 模板固定 + 每次附带 `assets_hd2d/style/style-board.png` 作风格锚定参考
2. **后处理**（PIL）：去 #ff00ff 背景 → nearest 缩放到规范尺寸 → 量化到全局调色板
3. **自动质检**：尺寸校验 / 透明边距校验 / 调色板偏差报警
4. **入库**：`assets_hd2d/{type}/{id}.png`，命名规范化

> 这条管线是后面扩到 10-15 个社群角色的前提——新群友加入，跑一遍管线即出图。

### 3.3 重生成清单

- [ ] 角色 ×3 按新规格重做（保留现有 prompt 的人设描述）
- [ ] 建筑 ×5 同角度同比例重做（单独出图，不再一行 5 个合排）
- [ ] 地形 tile 集（替换单张大纹理）
- [ ] 装饰物：油桶 / 木箱 / 晾衣绳 / 篝火 / 围栏（地图层次的关键，现在完全没有）

## 四、渲染升级：HD-2D 三件套（P0.5）

1. **真深度**：去掉 `no_depth_test`；billboard 改 Y 轴固定（`BILLBOARD_FIXED_Y`），角色能走到建筑后面被遮挡
2. **影子**：每个 sprite 脚下加椭圆渐变贴花（quad + 径向渐变纹理，跟随移动）；DirectionalLight 开 shadow
3. **tilt-shift**：全屏 ColorRect + screen texture shader，上下各 25% 渐进高斯模糊（compatibility renderer 可用，不依赖 DoF）
4. **昼夜循环**：平行光角度 + 色温随游戏时刻 lerp（清晨橙 → 正午白 → 黄昏红 → 夜蓝紫）；夜间建筑窗户 OmniLight 点亮、篝火光闪烁
5. **glow**：environment glow（Godot 4.3+ compatibility 已支持；不行则 shader 模拟高光溢出）
6. **地面重做**：多 quad tile 拼接（或 GridMap），2-3 种变体随机分布，道路用 tile 而非 BoxMesh

## 五、手机 UI/交互重构（P0）

### 5.1 分辨率策略（一次改对，全局生效）

- 设计分辨率改 **432×936**（手机逻辑像素级），stretch 保持 `canvas_items/expand`
- 字号基准：正文 15-17，标题 18-20；按钮高度 ≥48
- safe-area 适配（刘海/底部手势条）：读 `DisplayServer.get_display_safe_area()` 偏移 UI

### 5.2 首屏只留三件东西

```
┌──────────────────────────┐
│ 第2天 18:26  🍞42 💧58 ⚡31 ❤️18 😊64 ⚠️37 │ ← 一条紧凑状态条（图标+数字）
│                          │
│                          │
│        世界 ~80%          │ ← 拖拽/双指缩放
│   （角色头顶对话气泡）       │
│                          │
│                          │
├──────────────────────────┤
│   今日    │   投票   │  营地  │ ← 底部 dock，3 tab
└──────────────────────────┘
```

### 5.3 交互范式：bottom sheet 统一所有内容面板

- 每日简报、人物卡、建筑卡、投票、营地历史 → 全部从底部滑出**半屏 sheet**，点外部关闭，下滑收起
- 人物卡升级：头像大图（用 sheet 第一帧放大）+ 状态行 + 记忆时间线
- 投票：选项点击乐观更新 + 票数动画；已投状态明显（高亮+✓）

### 5.4 台词与事件

- 角色台词 → 头顶气泡（Label3D，挂在现有名字标签下方），4 秒淡出；删除全局 speech 条
- 事件 → 顶部滑入 toast，3 秒消失；全部事件落入「今日」tab 的时间线，可回看

### 5.5 时间映射（产品级关键决定）

**默认"实时模式"：游戏内时刻 = 北京时间。**
早上打开看到角色在厨房吃早饭，晚上打开看到掩体亮灯——这才是"日更活世界"的正确体验，而不是 2 分钟一天的快进循环动画。

- 「今日」tab 提供 **回放** 按钮：从 06:00 快进重看今天的剧本（保留现有 time_speed 逻辑）
- 暂停/倍速只在回放模式下出现，默认界面不出现

### 5.6 触控修正

- 点击判定：UI 面板开启时拦截世界点击（`set_input_as_handled`）；角色选中半径按屏幕密度换算
- 单指拖拽平移 / 双指捏合缩放保留，加缩放边界回弹

## 七、画面基准修订：整图底板路线（2026-06-11，Major 以 style-board.png 定标）

> Major 明确：界面感受至少要达到 `assets_hd2d/style/style-board.png` 的水准。
> 结论：瓦片+单体 sprite 的模块化拼装在结构上达不到该密度，切换为**整图底板**路线。

### 7.1 路线

- **静态层**：地形+建筑+围栏/菜畦/碎石等装饰 → 直接生成 1 张"无人物完整营地底板"大图（竖屏 9:19.5 构图），替换现有 tile 地面 + 建筑 sprite
- **动态层**：仅角色、篝火动画、炊烟粒子、窗灯、头顶气泡作为 sprite 叠加
- **遮挡**：角色只走小路（日程点可控）；需要被挡的位置从底板抠 2-3 块建筑前景切片做 occluder
- **昼夜**：同一底板 img2img 出 晨/昼/黄昏/夜 4 时段版，按北京时间 crossfade + 加法光（窗灯/篝火）
- **建筑解锁**：Runner 日更时重出底板——"今天营地变样了"本身是核心体验
- 现有角色管线保留；角色重出时以底板为风格参考对齐密度与色板

### 7.2 竖屏相机三状态（默认不需要移动地图）

1. **全景（默认）**：一屏装下整个营地，打开即截图
2. **探索**：捏合放大后单指拖拽 + 边界回弹；双击回全景
3. **跟随**：人物卡"跟随"按钮，镜头拉近跟角色走日程
4. 未来扩张只沿纵轴（往北面废墟），地图变竖向长卷，单指上下滑

### 7.3 执行顺序

1. 黄昏版底板生成 + 替换渲染层（画面即达参考图 80%）
2. 角色按底板风格重出 + 动态层（炊烟/篝火/窗灯）
3. 昼夜 4 时段底板 + crossfade
4. 相机三状态
5. 回到主线：Runner 日更 + 群友化身（见 2026-06-11 体验分析结论）
